The division of memory

  • area of code:binary code, which is monitored by the operating system
  • area of variables:
  • stack
  • heap
before the program is executed

After the program is translated, exe excutable program is created. Two areas are created before the program is executed:

  • area of the code

    the opcode that is going to be operated by the machine

    this area is shared

    this area cannot be written

  • area of variables

    global variable and static variable are put here

    constant is stored here,too

    data in this area will be released by the operating system after the program is finished

after the program is executed

  • stack

    released by compiler, store variables in the function, local variable

    do not return the address of the local variable

  • heap

    released by the user

    Use “new” to start a memory

    delete the data: use “delete”

  • Add “[]” when releasing an array
