easy ways to sort a list
bubblesort
  • exchange two adjacent numbers in a list if the interior one is larger
  • after one loop of sorting is done, the area of disorder has one less number while the ordered area has one more
selection sort
insertion sort

insert the Nth element in a list in to the first n-1 numbers (arranged)

continue until all of the list is arranged

Complex
quick sort
heap sort

the process of heap sorting：

1.build a heap

2.obtain the element on the top of the heap,which is the largest element

3.forbid the top element and put the last element of the heap on to the top

4.the top element now is the second largest element

5.continuing 3 until the heap is empty

the problem of TOPK: try to figure out the largest K numbers in a list of n numbers

use heap sort to solve TOPK

1.find the first k numbers in a list to make a small heap

2.put away the top the heap(the currently kth smallest number)

3.compare the first number from k to n with the top

4.if smaller:move to the next number

if larger: put it on the top and sift it again

5.continue it

merge sort

a “merge” means two ordered list become one ordered list

the way of merging: compare the two lists from the first numbers

the way to use “merge”

  • Decompose the list until only one element is left

  • end condition: only one element is in ordered

  • Merge: use merge to merge lists continously

a conclution for the complex sorts

  • time complexity are all O(nlogn)

  • the time taken:

    Quick sort<merge sort<heap sort

  • the dosadvantages for the three alogrithnms

    quick sort: low efficienty in some extreme situation

    Merge sort: have space complexity

    heap sort: relatively slow

  • Stability:

    Stable:bubble sort, insert sort, merge sort
